# Zasady współpracy (Contributing)

Dziękujemy za zainteresowanie współpracą przy projekcie
„Przewodnik po stolicach Europy”.

## Nazewnictwo gałęzi (branchy)
Każda osoba pracująca nad projektem powinna utworzyć osobną gałąź
według schematu:

nazwa-stolicy-imie

Przykład:
- rzym-yulia

## Wiadomości commitów
Wiadomości commitów powinny być:
- krótkie
- zwięzłe
- jasno opisujące wprowadzone zmiany

## Proces pracy
1. Wykonaj fork repozytorium
2. Utwórz nową gałąź
3. Wprowadź zmiany w kodzie
4. Wyślij zmiany do repozytorium
5. Utwórz Pull Request do gałęzi main

## Komunikacja
Wszelka komunikacja dotycząca projektu powinna odbywać się
za pośrednictwem komentarzy w Pull Requestach lub zgłoszeń Issues
na GitHubie.
